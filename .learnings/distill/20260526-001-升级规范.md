# 教训：OpenClaw 升级的正确流程（2026-05-26）

## 问题
升级时直接删除旧版 node_modules 路径，导致飞书等插件丢失。

## 根因
1. 旧版插件装在 `/Users/mac/.local/lib/node_modules/openclaw/`
2. 新版装在 `/opt/homebrew/lib/node_modules/openclaw/`
3. 升级前没有确认插件安装位置，直接删了旧版路径
4. 重启 Gateway 时 plugin system 找不到飞书插件

## 正确做法
1. **升级前**：先查清楚插件安装位置
   - `npm root -g` 确认全局路径
   - `openclaw plugins list` 查所有插件及其位置
2. **升级前**：备份插件列表
3. **升级后**：验证所有插件状态，再重启 Gateway
4. **重启前**：确认新版路径包含所有插件
5. **不要直接删旧版**，等确认新版正常后再清理

## 正确升级命令顺序
```bash
# 1. 查当前插件状态
openclaw plugins list

# 2. 确认新版路径
npm root -g  # 应该是 /opt/homebrew/lib/node_modules

# 3. 安装新版
npm install -g openclaw@latest

# 4. 验证插件
openclaw plugins list | grep -E "feishu|email|enabled"

# 5. 确认插件位置是新版路径后再重启
```
