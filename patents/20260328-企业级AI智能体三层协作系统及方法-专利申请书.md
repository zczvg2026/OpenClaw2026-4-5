## 发明名称

**一种基于三层文件记忆架构的企业级AI智能体协作系统及方法**

---

## 技术领域

本发明属于人工智能技术领域，具体涉及一种通过文件系统实现多个AI智能体（Agent）长期协作的企业级方法架构。

---

## 背景技术

### 现有技术的不足

当前企业部署AI智能体的主要模式存在以下缺陷：

**1. 会话记忆无法跨周期积累**
现有AI Agent基于单次会话运行，对话结束后所有上下文丢失。每次启动均为"从零开始"，无法形成持续增长的能力积累。

**2. 多Agent协作依赖外部系统**
现有多Agent方案通常依赖消息队列、API调用或数据库作为协调层，系统复杂度高，部署成本大，且存在单点故障风险。

**3. 企业场景的上下文断裂**
企业运营涉及多角色（销售/运营/客服/管理层），各角色AI之间的信息不互通，无法形成统一的组织记忆，导致AI输出与企业实际需求脱节。

**4. 缺乏可持续演化的架构**
现有方案将AI能力固化在模型本身，而非固化在积累的上下文文件中。一旦更换模型，所有"经验"归零。

### 现有技术的典型方案

- **OpenAI/Claude的Assistant API**：基于向量数据库存储记忆，但需要额外的工程开发
- **CrewAI等框架**：基于代码层协调，依赖API调用
- **传统企业AI系统**：基于数据库+规则引擎，缺乏自适应演化能力

上述方案均未提出一套仅依赖文件系统即可实现、可积累、可演化的企业级多Agent协作方法。

## 发明内容

### 核心创新点

本发明提出一种**三层文件记忆架构**，通过文件系统作为集成层，实现AI智能体的长期协作与组织记忆积累，无需数据库、消息队列或复杂中间件。

### 系统架构

```
┌─────────────────────────────────────────────────────┐
│                   组织AI智能体系统                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  第一层：身份层（Identity Layer）                    │
│  ├── SOUL.md      — 智能体角色定位与行为性格           │
│  ├── IDENTITY.md  — 快速参考名片（名字/角色/风格）    │
│  └── USER.md      — 服务对象画像与偏好               │
│                                                     │
│  第二层：操作层（Operations Layer）                   │
│  ├── AGENTS.md   — 启动流程与行为规则               │
│  ├── HEARTBEAT.md — 自检监控与健康监控              │
│  └── 专业规则文件 — 角色专属操作指南                 │
│                                                     │
│  第三层：知识层（Knowledge Layer）                   │
│  ├── MEMORY.md           — 长期记忆（精炼版）         │
│  ├── memory/YYYY-MM-DD.md — 每日操作日志            │
│  └── shared-context/    — 跨智能体共享上下文         │
│       ├── THESIS.md     — 组织当前战略世界观          │
│       ├── FEEDBACK-LOG.md — 跨角色纠错记录          │
│       └── SIGNALS.md    — 外部信号与趋势追踪         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 核心方法步骤

**步骤一：初始化组织级身份层**
为每个AI智能体配置SOUL.md定义其角色定位与行为原则，配置USER.md定义其服务对象的背景与偏好。

**步骤二：建立共享上下文中枢**
创建shared-context/目录作为跨智能体共享的单一真相来源，其中：
- THESIS.md由管理层智能体维护，记录组织的战略方向与核心假设；
- FEEDBACK-LOG.md记录跨智能体纠正，所有智能体读此文件避免重复错误；
- SIGNALS.md记录外部环境变化，供所有智能体感知市场动态。

**步骤三：构建前后端双层协作结构**
- **后端智能体**（策略中枢）：负责制定战略、分析数据、更新THESIS.md和SIGNALS.md
- **前端智能体**（执行终端）：负责具体任务执行，读共享上下文，按后端指令行动
- 两者通过共享文件交接，以单写者规则避免冲突

**步骤四：建立时间积累机制**
- 每日操作日志写入memory/YYYY-MM-DD.md
- 定期将高频纠错和重要决策提炼至MEMORY.md
- MEMORY.md在每次会话启动时加载，实现"经验"的跨周期传递

**步骤五：心跳自检与系统自演化**
通过HEARTBEAT.md定期检查各智能体的运行状态和任务执行情况，捕获静默失败，自动触发修复。

### 技术效果

| 指标 | 现有技术 | 本发明 |
|------|---------|-------|
| 多Agent协作依赖 | 消息队列/API | 文件系统（零依赖）|
| 上下文积累 | 会话结束丢失 | 永久文件积累 |
| 经验复用 | 每次从零开始 | 提炼入MEMORY.md |
| 跨角色纠错 | 需要重复告知 | FEEDBACK-LOG一次记录 |
| 新Agent启动 | 需要完整重新配置 | 读取共享上下文即对齐 |
| 护城河积累 | 依附于模型 | 依附于文件，可跨模型迁移 |

## 具体实施方式

### 实施例一：企业营销团队场景

**背景：** 某招商加盟企业部署本系统，包含：
- 管理层智能体（大龙虾·后端）
- 获客智能体（小龙虾A）
- 客服智能体（小龙虾B）
- 内容智能体（小龙虾C）

**实施步骤：**

1. 管理层智能体扫描市场数据，更新`shared-context/SIGNALS.md`，记录"某竞品推出新功能"
2. 管理层智能体分析后更新`shared-context/THESIS.md`，将"竞品应对"写入战略方向
3. 获客智能体读`shared-context/`后，自动生成针对竞品的获客话术
4. 客服智能体读`shared-context/`后，自动更新FAQ应对竞品质疑
5. 内容智能体根据`shared-context/`生成对比内容

所有纠正在`FEEDBACK-LOG.md`记录一次，三条小龙虾均读取并修正，避免同一错误重复出现。

### 实施例二：个人知识管理者场景

**背景：** 知识工作者部署单主智能体+多个辅助智能体

**实施步骤：**

1. 研究智能体扫描行业信息，写入`intel/DAILY-INTEL.md`
2. 写作智能体读`intel/DAILY-INTEL.md`，生成文章草稿
3. 主智能体审核草稿，给出纠正，写入`FEEDBACK-LOG.md`
4. 写作智能体在下一次启动时自动加载`FEEDBACK-LOG.md`，无需重复告知

## 权利要求书

### 独立权利要求

**1.** 一种基于文件系统实现的多智能体协作方法，其特征在于，包括：
   - 为每个智能体配置身份定义文件，建立角色定位；
   - 构建共享上下文中枢，包含战略世界观文件、跨智能体纠错日志文件和环境信号追踪文件；
   - 所述共享上下文中枢被所有智能体在每次启动时读取；
   - 多个智能体之间通过文件读写进行任务交接，以单写者规则避免写入冲突。

**2.** 一种企业级AI智能体协作系统，包括：
   - 第一层身份定义模块，用于配置各智能体的角色定位、服务对象画像和行为规则；
   - 第二层操作控制模块，用于定义各智能体的工作流程和启动检查规则；
   - 第三层知识积累模块，包括长期记忆文件、每日操作日志和共享上下文中枢；
   - 所述共享上下文中枢包含战略世界观文件、跨智能体纠错日志文件和环境信号追踪文件；
   - 各智能体通过读取所述共享上下文中枢实现跨周期经验积累。

### 从属权利要求

**3.** 根据权利要求1所述的方法，其特征在于，所述长期记忆文件在每次会话启动时自动加载，内容包括经提炼的重要决策、业务教训和持续优化的行为规则。

**4.** 根据权利要求1所述的方法，其特征在于，所述每日操作日志以日期命名，定期由人工或智能体将高频纠错和重要经验提炼至所述长期记忆文件。

**5.** 根据权利要求1所述的方法，其特征在于，还包括心跳自检模块，用于定时检查各智能体的任务执行状态，在检测到静默失败时自动触发修复流程。

**6.** 根据权利要求2所述的系统，其特征在于，所述系统部署于企业运营场景，包括后端策略智能体和至少一个前端执行智能体，所述后端智能体负责更新所述战略世界观文件，所述前端执行智能体依据所述战略世界观文件执行具体任务。

**7.** 根据权利要求6所述的系统，其特征在于，所述后端策略智能体与所述前端执行智能体之间不存在直接的API调用或消息队列通信，仅通过文件系统中的共享文件进行任务交接。

## 摘要

本发明公开了一种基于三层文件记忆架构的企业级AI智能体协作系统及方法。系统由身份层（定义智能体角色）、操作层（定义工作流程）和知识层（积累组织记忆）三层构成，通过文件系统作为集成层实现多智能体的跨周期协作与经验积累。共享上下文中枢作为单一真相来源，包含战略世界观、跨智能体纠错日志和环境信号追踪文件，确保所有智能体对齐同一组织目标。本发明无需数据库、消息队列等复杂中间件，仅通过文件读写实现零依赖协作，使组织记忆随时间持续积累，形成难以复制的AI能力护城河。
