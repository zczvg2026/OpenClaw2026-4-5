# Claude Code 泄露事件 · 关键结论存档

## 事件背景

**2026年3月31日**，Claude Code（Anthropic 官方 AI 编程工具）的完整源码被泄露，瞬间传遍开发者社区。泄露的是 Claude Code 的 TypeScript 源码压缩包，约1423个 `.ts` 文件，包含完整实现。

## 源码研究结论

### 原版 vs claw-code（改写版）对比

| | 原版 (claude-code-complete) | 改写版 (claw-code / instructkr) |
|--|----------------------------|----------------------------------|
| 语言 | TypeScript | Python |
| 文件数 | 1423 个 .ts | 9 个 .py |
| 完成度 | 完整生产级 | 仅骨架，不可执行 |
| Stars | — | 50K+（2小时破纪录） |
| 法律状态 | 泄露作品，风险高 | clean-room 重写，法律干净 |

**改写版 claw-code 实际上是什么：**
- 元数据索引器（工具/命令清单的 Python 包装）
- 架构文档生成器
- Python 学习项目
- **不是 Claude Code 的替代品，不具备实际执行能力**

### 原版 Claude Code 的核心模块（按价值权重排序）

1. **权限分类引擎**（Permission System）⭐⭐⭐⭐⭐
   - 五层判断：规则锁→路径锁→命令分析→沙箱→人类确认
   - 最值钱的部分：AI 判断 AI 行为的 auto 模式
   - 企业买单的核心原因

2. **工具系统**（Tool System）⭐⭐⭐⭐
   - 50+ 内置工具，读写执行搜索等
   - 关键是让 AI 自己判断用哪个工具、怎么组合

3. **上下文管理**（Context Management）⭐⭐⭐
   - 对话压缩、记忆提取、token 预算
   - 决定 AI 能处理多复杂的任务

4. **远程桥接**（Remote Bridge）⭐⭐
   - Session 生命周期、多会话并发、崩溃恢复
   - 工程量最大，护城河最深

5. **多 Agent 协调**⭐
   - 还在早期，但决定终局

## 建议（供太擎/域擎参考）

**不推荐做的事（市场化基因）：**
- 从头重写 Claude Code（研究驱动，18个月起步）
- 直接商业化源码（法律风险）
- 复制工具链（OpenClaw 已有类似能力）

**推荐做的事：**
1. 作为技术选型的"对标锚点"——知道差距在哪里
2. 研究权限系统设计，作为给大客户讲的"安全故事"
3. 判断 OpenClaw 哪些功能直接用、哪些自己补：
   - OpenClaw 已有：工作流编排、基础 Agent、多模型接入、工具系统
   - 需要自己补：权限分类引擎、企业审计日志、私有化部署、客户权限后台

**结论：权限系统不需要从头写，但需要认真做。这是签企业大单的前提。**
