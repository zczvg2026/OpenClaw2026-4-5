# 2026-03-29 日志

## 今日主要工作

### 飞书掉线问题排查与修复
- **根本原因**：双 LaunchAgent 冲突
  - `ai.openclaw.gateway`（大闸蟹）+ `com.openclaw.gateway`（Q小倩）同时注册
  - 每次 SIGUSR1 重启时两个服务互相抢占，飞书 WebSocket 被中断
- **修复措施**：
  1. 停用了多余的 `com.openclaw.gateway` plist（移除了 RunAtLoad/KeepAlive）
  2. Q小倩的 plist 路径后来已修正（但Q小倩 Gateway 由 QClaw.app 内嵌启动，非 plist）

### 大闸蟹 vs Q小倩 冲突分析（完整版）
- 端口：18789 vs 28789 ✅ 无冲突
- 飞书 BotId：cli_a93eaef... vs 环境变量 `${QCLAW_FEISHU_APP_ID}` ✅ 无冲突
- 配置文件：~/.openclaw/ vs ~/Library/Application Support/QClaw/openclaw/ ✅ 无冲突
- **结论：两套实例可和平共处，Q小倩 Gateway 内嵌在 QClaw.app 里**

### WeChat 报错问题修复
- Q小倩的 `wechat-access` 插件在不断报错 `fetch failed` / `ENOTFOUND open.feishu.cn`
- 已禁用：`plugins.entries.wechat-access.enabled = false`（Q小倩和我的配置都禁了）
- WeChat 报错已完全消失
