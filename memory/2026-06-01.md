# 大闸蟹每日日记 - 2026-06-01

## 今日最重要的一件事
**搞定飞书 user_access_token OAuth 授权**，能以 Johnson 个人身份读所有 34 个飞书群。

## 主要工作内容
- **OAuth 授权脚本搭建**：创 `.feishu-user/` 目录（700 权限），存凭证 + token
- **踩 5 个坑**（详见 `.learnings/distill/20260601-001-feishu-oauth-v1-app-id-params.md`）：
  1. v2 路径全 404（v1 才是对的）
  2. Token 端点 v2 404（v1 才是对的）
  3. JSON body 不是 form
  4. `app_id` / `app_secret` 不是 `client_id`
  5. 响应包在 `data` 里
- **全量拉群成功**：34 个群，存档到 `memory/feishu-chats/chats.json`
- **关键发现**：飞书文档站 JS 渲染，web_fetch 拿不到正确路径，**用 curl 试 endpoint 状态码最快**（v1 200 / v2 404）

## 今日教训
- 飞书 API 响应统一是 `{code, msg, data}` 三件套
- 飞书不守 OAuth 标准，自己用 `app_id` / `app_secret`
- OAuth code 一次性，调试时多次跑需要每次拿新 code
