# 域擎课程核心原则（Johnson 2026-03-28 确立）

## 总纲

> **以 AI 为中心组织生产力，而不是以人为中心。**
> 人类从"执行者"变成"验收者"，AI 从"工具"变成"主力"。

---

## 六条组织原则

**原则1：上下文即资产**
- 全部沟通在飞书，项目任务化，一张表
- 对齐上下文，最大化上下文
- 上下文越充分，AI输出质量越高

**原则2：用最好的模型**
- 暂时不要怕 token 消耗
- 便宜模型省的钱，远远抵不上质量损失
- 用最好模型，拿到最直接的输出

**原则3：全员 AI，反向指导**
- 用 AI 反向指导工作，而不是我们指导它
- 人是 AI 的验收者和调校者，不是指挥者

**原则4：SOP 的作用反转**
- SOP 的框架仍然需要，但作用变了
- 以前：SOP 是为了约束人执行（省人力）
- 现在：SOP 是为了给 AI 提供稳定输入格式（省 token 仍有用）
- 核心：给足 token，AI 直接拿结果

**原则5：表达视频化**
- 人类已经读不了长文字了，PPT 已落后
- 所有想表达的，都做成视频
- 目的：省去用户学习成本，直达认知

**原则6：AI 替代老师**
- 所有场景，都考虑用 AI 解答疑问
- 把老师的经验变成可复用的 AI 系统
- 不再依赖个人经验传承

---

## 底层逻辑

这六条共同指向一个核心转变：

```
旧模式：人执行 → AI 辅助
新模式：AI 执行 → 人验收
```

**所有标准化工作流的终点，是让 AI 能稳定产出，人来做最后判断。**
