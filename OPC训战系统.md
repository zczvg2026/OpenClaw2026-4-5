# OPC 训战系统 v2

> OPC = One Person Company（一人公司）
> 目标：让一个人用AI，独立完成从前到后的全链路

---

## 核心理念

**一个人就是一个公司。** 不需要团队、不需要开会、不需要管理成本。
AI 是员工，你是 CEO+PM+销售+交付。

旧模式：人执行 → AI 辅助
**新模式：AI 执行 → 人验收**

---

## 训（培训）— 3 堂课，每堂30分钟

### 第1课：一人公司的武器库
- 你有哪些武器？—— 大模型、智能体、自动化工具
- 核心原则：上下文即资产，所有沟通进飞书
- 一个比喻：你不是在"用AI"，你是在"组建一个虚拟团队"
- **交付**：一张武器清单（装什么工具、用来干什么）

### 第2课：一人公司的销售怎么打
- 拓客 → 触达 → 跟进 → 成交，全流程AI化
- **关键转变**：AI替你写文案、发消息、管线索，你只做判断和决策
- 场景演练：一小时内完成100个客户的触达
- **交付**：一条AI销售流水线（从获客到成交的prompt链）

### 第3课：一人公司的交付怎么跑
- 服务/产品交付全流程AI化
- 核心：SOP 不是约束你的，是给AI的输入格式
- 常见坑：什么事都要自己做（戒掉）、什么事都丢给AI（也不行）
- **交付**：一条交付流水线模板

---

## 战（实战）— 3 场战役

### 第1战：24小时从0到1（单人作战）
- **目标**：24小时内，选一个方向，做一个可展示的东西出来
- **流程**：选方向 → AI出方案 → 快速验证 → 出成品
- **复盘**：哪里卡住了？AI的哪个环节拉胯了？人的判断哪里慢了？

### 第2战：真实成交跑一圈（全链路）
- **目标**：从获客到成交，真实跑一遍
- **流程**：找100个目标客户 → AI批量触达 → 跟进 → 至少拿到1个有效沟通
- **复盘**：转化率？哪个环节断了？AI写的文案被打回来几次？

### 第3战：交付一条龙（服务闭环）
- **目标**：把一个真实客户/项目从签单做到交付
- **流程**：需求确认 → AI出方案 → 人验收 → 交付 → 收款
- **复盘**：交付质量？客户满意度？下次能不能更快？

---

## 配套工具

| 工具 | 内容 |
|------|------|
| 武器清单 | AI工具链一览表（大模型/智能体/自动化） |
| 销售流水线 | 获客→触达→跟进→成交的prompt链 |
| 交付流水线 | 签单→需求→方案→交付的SOP模板 |
| 复盘模板 | 每场战役后的标准化复盘问题卡 |

---

## 节奏

**Day 1**：3堂课（1.5小时）+ 第1战 24小时挑战开始
**Day 2**：第1战复盘（30分钟）+ 第2战开始（真实成交跑一圈，持续2-3天）
**Day 3-5**：第2战实战 + 复盘
**Day 6-7**：第3战交付闭环 + 复盘总结

---

## 铁律

1. **不做PPT** — 张嘴就说
2. **不要教理论** — 只教能用得上的
3. **战 > 训** — 实战时间至少培训2倍
4. **每战必复盘** — 不复盘=白练
5. **AI全程参与** — 培训本身也用AI加速
