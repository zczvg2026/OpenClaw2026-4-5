# 🦀 OPC积分商城小程序 — 任务包

## 项目背景
张承中（Johnson）要做一个积分商城微信小程序，带4级会员体系和4级分销网络。

## 需求文档（已确认）

**产品定位**：积分商城，只收积分（1元=1积分），不收现金。

**商品**：实物 + 虚拟服务（课程/会员身份/书籍/Token包/AI智能体手机/各类智能体）

**会员体系（4级）**：
- OPC：注册即得，基础采购价
- OPC工会：发展5个OPC升级，更低价采购
- OPC社区：发展2个OPC工会升级，更低价采购
- OPC城市：一年内完成300万业绩（审核制），最优惠采购价

**分销网络（4级佣金）**：
- 扫码注册，终身绑定上下级
- A→B→C→D→D均有佣金，佣金来自等级差价格利润

**交易规则**：
- 积分充值购物，1:1现金充值
- 各商品单独设计积分价格
- 各等级对应不同采购积分价

**开发目标**：1个月内上线MVP

## 技术方案

**后端**：MedusaJS v2（Node.js + TypeScript + PostgreSQL）
- 参考项目：https://github.com/mercurjs/mercur
- 重点开发：积分钱包模块 + 分销关系模块（Mercur多商户订单分单功能已完整，直接复用）

**数据库扩展（9张表）**：
- MemberLevel（会员等级配置）
- Member（会员主表，1:1关联MedusaJS Customer）
- DistributionNode（分销关系表）
- DistributionPath（链路快照表，预计算3层）
- PointAccount（积分账户）
- PointLedger（积分流水，不可变日志）
- ProductLevelPrice（商品等级价格矩阵）
- LevelAdvancement（晋升记录）
- InviteCode（邀请码，扫码绑定入口）

**前端**：微信小程序
- 技术选型：Taro + Vue 3 + TypeScript
- 核心：扫码绑定、积分商城、会员中心、分销佣金查看

**合规**：分润最多3层（中国法律限制），Level4城市等级仅作身份标签
