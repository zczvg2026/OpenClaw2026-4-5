# 凤凰归巢策略规则

## 总流程

1. 模式识别
2. 情绪判断
3. 板块筛选
4. 龙头识别
5. 二连板检测
6. 回调结构判断
7. 模型分类
8. 模式化买点识别
9. 模式化信号评分
10. 自动选股
11. 仓位分配
12. 模式化执行动作
13. 风控监控
14. 动态止盈
15. 异常过滤
16. 持仓管理
17. 信号输出
18. 优先级控制

## 模式识别

支持三种模式：

- `pre_market`：用于 09:30 前数据，核心任务是生成开盘观察名单
- `intraday`：用于盘中实时快照，核心任务是确认是否可执行
- `post_market`：用于收盘后数据，核心任务是生成次日观察池和持仓处理结论

脚本优先读取显式模式字段，其次可根据 `snapshot_time` 推断：

- `< 09:30`：`pre_market`
- `09:30 ~ 15:00`：`intraday`
- `>= 15:00`：`post_market`

## 情绪判断

- 冰点：涨停数 < 20，禁止交易
- 震荡：涨停数 20~50，仅做强信号
- 上升：涨停数 > 50，全策略执行
- 退潮：高标跌停，禁止追高

## 板块筛选

- 仅保留涨停数 Top3 板块
- 板块必须存在连板股

## 龙头识别

按以下顺序择优：

1. 连板高度最高
2. 最早涨停
3. 成交额最大

非龙头仓位减半。

## 前置条件

- 候选股必须满足连续 2 个涨停

## 洗盘与回调结构

出现以下任意情况即视为无效：

- 二板后跌停
- 单日振幅 > 15%
- 连续两日长阴
- 两日跌幅 > 15%
- 放量下跌
- 高位加速

## 模型分类

- 金凤凰：回调低点 >= 二板高点
- 银凤凰：回调低点 >= 首板高点
- 火凤凰：回调低点 >= 首板 1/2 位

首板 1/2 位优先使用显式字段；没有显式字段时，可用首板最高价与最低价的中位数代替。

## 独立规则域

### pre_market

只使用竞价和昨日日线信息。

- 允许：
  - 竞价强度评分
  - 开盘观察名单排序
  - 计划仓位建议
- 禁止：
  - 输出 `buy_signal`
  - 输出即时开仓结论

观察优先级量化：

- S 级：高开强势 + 竞价量比强 + 封单稳定
- A 级：高开明显 + 竞价放量
- B 级：小幅高开 + 板块龙头加分
- C 级：仅情绪上升期允许，作为备选观察

输出类型：

- `watchlist`

### intraday

只使用盘中可确认的行为数据。

- 允许：
  - 输出 `buy_signal`
  - 输出 `entry_price`
  - 输出 `stop_loss`
  - 输出执行方式
- 禁止：
  - 输出盘后复盘结论
  - 输出长期统计类判断

盘中买点量化：

- S 级：倍量柱 + 缩量整理 + 放量突破 + 回封确认
- A 级：放量突破调整高点
- B 级：MA10 支撑 + 阳线反包
- C 级：仅情绪上升期允许

输出类型：

- `buy_signal`

### post_market

只使用收盘后确认信息。

- 允许：
  - 复盘结论
  - 次日观察池
  - 持仓处理建议
- 禁止：
  - 输出即时买点
  - 输出即时开仓建议

盘后复盘量化：

- S 级：放量突破后收盘站稳 + 收盘接近高位
- A 级：突破调整高点且收盘强势
- B 级：MA10 支撑 + 阳线反包
- C 级：仅情绪上升期允许，作为次日低优先级观察

输出类型：

- `review`

## 评分模型

### pre_market

综合评分 = 情绪 35% + 板块 25% + 龙头 20% + 形态 10% + 买点 10%

### intraday

综合评分 = 情绪 30% + 板块 25% + 龙头 20% + 形态 15% + 买点 10%

### post_market

综合评分 = 情绪 25% + 板块 25% + 龙头 20% + 形态 20% + 买点 10%

基础量化口径：

- 情绪：冰点 0，退潮 0，震荡 65，上升 100
- 板块：Top1 100，Top2 85，Top3 70，其余 0
- 龙头：龙头 100，非龙头 50
- 形态：金凤凰 100，银凤凰 85，火凤凰 70
- 买点：S 100，A 85，B 70，C 55

## 仓位分配

- S 级：50%
- A 级：30%
- B 级：20%
- C 级：10% 以内

在以下情况进一步收缩仓位：

- 非龙头：基础仓位减半
- 震荡期：基础仓位再减半
- `pre_market`：仅给计划仓位，不给直接开仓结论
- `post_market`：仅给次日计划仓位，不给当下直接开仓结论

## 入场规则

### pre_market

- 输出开盘盯盘优先级
- 不直接视为盘前成交指令
- 行为建议以“开盘后观察”为主

### intraday

- 突破时跟随
- 回封确认后跟随
- 尾盘确认进场
- 涨停板挂单

### post_market

- 不输出即时买入结论
- 输出次日观察池、明日关注价与持仓处理建议

脚本默认优先使用 `breakout_price` 作为建议入场价，没有时退回 `current_price`。

## 止损规则

优先级从高到低：

1. 上攻 K 线低点
2. 首板收盘价
3. 首板 1/2 位

## 动态止盈

任一条件触发即提示减仓或退出：

- 放量滞涨
- 高位上影
- 跌破 MA5
- 龙头断板
- 情绪退潮

## 输出优先级

始终按以下顺序解释决策：

1. 情绪
2. 板块
3. 龙头
4. 形态
5. 模式独立输出
